# Security Policy

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities, exposed secrets, or private user-data risks.

Report security concerns through GitHub's private vulnerability reporting for this repository when available. If that is not available, contact the repository owner privately and include:

- the affected area or file
- steps to reproduce
- expected impact
- any safe proof of concept that does not expose real user data or secrets

The app must not commit signing certificates, provisioning profiles, OAuth client secrets, API keys, HealthKit exports, CloudKit production data, or user nutrition/body data.
