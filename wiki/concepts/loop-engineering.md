# Loop Engineering

Loop engineering means designing repeatable agent systems rather than one-off prompts.

A useful loop has:

- a clear goal
- a sensing step
- an action step
- an independent verification step
- a memory update
- a stopping condition
- a budget or cadence limit

Failure mode: an unattended loop can compound mistakes. Keep destructive actions permissioned, keep checks independent, and write down durable discoveries.
